## Status

**Ready for Review**

## Story

**As a** B2B 잠재 고객 (웹사이트 방문자),
**I want** 대동의 제품 정보를 보여주는 기본 홈페이지에 접속하고,
**so that** 회사와 제품에 대한 첫인상을 받고 주요 제품 카테고리를 탐색할 수 있다.

---

## Acceptance Criteria

1. **프로젝트 초기화**: Next.js 15 + TypeScript + Tailwind CSS v4 프로젝트가 생성된다
2. **shadcn/ui 설정**: shadcn/ui 컴포넌트 라이브러리가 설치 및 구성된다
3. **홈페이지 레이아웃**: 헤더(네비게이션), 메인 컨텐츠 영역, 푸터를 포함한 기본 레이아웃이 구현된다
4. **반응형 디자인**: 데스크톱, 태블릿, 모바일에서 올바르게 표시된다
5. **히어로 섹션**: 회사 소개 및 주요 메시지를 표시하는 히어로 섹션이 있다
6. **제품 카테고리**: 3M 제품, 전동공구 등 주요 카테고리 카드가 표시된다
7. **네비게이션**: 홈, 제품, 회사소개, 문의하기 메뉴가 작동한다
8. **타이포그래피**: 일관된 폰트 시스템과 컬러 팔레트가 적용된다
9. **SEO 기본**: 메타 태그, 타이틀, description이 설정된다
10. **빌드 성공**: `pnpm build` 명령이 에러 없이 완료된다

### 컴포넌트 패턴

**파일 명명 규칙:**
- 컴포넌트: PascalCase (예: `Header.tsx`, `HeroSection.tsx`)
- 절대 import 사용: `@/components/ui/button`
- Client Component는 파일 상단에 `'use client'` 지시어 필요

**컴포넌트 구조 예시:**
```typescript
'use client'; // 상호작용이 필요한 경우

import { Button } from '@/components/ui/button';

export function Header() {
  // 컴포넌트 로직
  return (
    <header>
      {/* JSX */}
    </header>
  );
}
```

### Next.js 15 App Router 규칙

- `app/(marketing)/page.tsx` → `/` 경로 (홈페이지)
- `app/(marketing)/layout.tsx` → 마케팅 페이지용 공유 레이아웃
- `app/layout.tsx` → 루트 레이아웃 (모든 페이지)
- Route Group `(marketing)` → URL에 나타나지 않음

### shadcn/ui 설치 방법

```bash
# CLI 초기화
npx shadcn@latest init

# 컴포넌트 추가
npx shadcn@latest add button
npx shadcn@latest add card
```


### 중요 주의사항

1. **타입 안전성**: 모든 컴포넌트는 TypeScript로 작성
2. **반응형 디자인**: Tailwind 브레이크포인트 사용 (`sm:`, `md:`, `lg:`, `xl:`)
3. **접근성**: 시맨틱 HTML과 ARIA 레이블 사용
4. **SEO**: 메타데이터는 `layout.tsx` 또는 `page.tsx`에서 export
5. **절대 경로**: `@/` 별칭 사용 (상대 경로 금지)

### 코딩 표준 (docs/architecture/coding-standards.md 참조)

**Critical Fullstack Rules:**
- **Component Imports**: 절대 import 사용 (`@/components/ui/button`)
- **Naming Conventions**:
  - Components: PascalCase (`ChatbotWidget.tsx`)
  - Files: kebab-case for pages, PascalCase for components
- **Type Safety**: TypeScript 엄격 모드, 모든 컴포넌트에 타입 정의

### 테스트 표준

**1. 컴포넌트 유닛 테스트 예시:**
```typescript
// tests/unit/components/layout/header.test.tsx
import { render, screen } from '@testing-library/react';
import { describe, it, expect } from 'vitest';
import { Header } from '@/components/layout/header';

describe('Header', () => {
  it('should render navigation menu', () => {
    render(<Header />);
    expect(screen.getByRole('navigation')).toBeInTheDocument();
    expect(screen.getByText('홈')).toBeInTheDocument();
    expect(screen.getByText('제품')).toBeInTheDocument();
  });

  it('should be responsive', () => {
    render(<Header />);
    // 모바일 메뉴 버튼 존재 확인
    expect(screen.getByLabelText('메뉴')).toBeInTheDocument();
  });
});
```

**2. E2E 테스트 예시:**
```typescript
// tests/e2e/homepage.spec.ts
import { test, expect } from '@playwright/test';

test('homepage should load correctly', async ({ page }) => {
  await page.goto('http://localhost:3000');

  // Hero section 확인
  await expect(page.locator('h1')).toBeVisible();

  // Navigation 확인
  await expect(page.locator('nav')).toBeVisible();

  // Product categories 확인
  await expect(page.locator('text=3M 제품')).toBeVisible();
});

test('should be responsive', async ({ page }) => {
  await page.setViewportSize({ width: 375, height: 667 }); // Mobile
  await page.goto('http://localhost:3000');

  // 모바일 메뉴 확인
  await expect(page.locator('[aria-label="메뉴"]')).toBeVisible();
});
```
